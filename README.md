## 项目简介

《背包幸存者》是一款融合 **Roguelike、幸存者、搜刮掉落与网格背包构筑** 的 3D 俯视角生存射击游戏。

玩家将进入持续恶化的封锁区，在敌群与区域威胁中移动、战斗和搜索物资，并通过有限的背包空间完成装备摆放、合成与联动构筑，逐步提升战斗能力，最终根据生存时间、击杀数量与战利品价值进行结算。

本项目不是对某款现有游戏的简单复刻，而是基于个人对“幸存者战斗节奏”和“背包构筑策略”的理解所提出的一套玩法方案。项目从游戏设计文档、系统拆分、工程结构到功能实现均由个人主导，主要用于验证玩法想法，并作为求职作品展示 Unity 客户端开发能力。

## 核心玩法构想

- **主动与自动双战斗系统**：主动射击提供操作空间，自动武器负责持续输出，两种战斗方式共享统一的伤害结算逻辑。
- **网格背包构筑**：背包不仅用于收纳，物品的占格、位置、合并和相邻关系会影响战斗能力。
- **战斗、搜刮与成长循环**：击败敌人获得掉落，通过拾取和整理背包形成新构筑，再挑战更高强度的波次。
- **区域压力与事件争夺**：污染区、补给点和限时事件推动玩家移动，减少固定站位带来的重复感。
- **单局积分结算**：以存活时间、击杀表现、事件完成度和战利品价值记录每局成果。

计划中的单局核心循环：

```text
进入关卡
  → 探索封锁区与清理敌群
  → 获取经验、装备和事件奖励
  → 整理背包、合并物品与调整构筑
  → 应对更高强度波次
  → 击败 Boss 或存活至结算
```

## 工程设计

项目采用按职责拆分的模块化结构，尽量将玩法数据、运行时规则和表现层分离，避免所有逻辑集中在单一管理器中。

| 模块 | 主要职责 | 设计方向 |
| --- | --- | --- |
| Core | 通用事件、时间、对象池和工具 | 为各玩法模块提供基础服务 |
| Data | 武器、敌人、物品、波次配置 | 使用 ScriptableObject 保存静态配置 |
| Gameplay | 玩家、战斗、敌人、掉落和波次 | 按功能拆分并通过接口协作 |
| Inventory | 背包网格与构筑规则 | 纯数据逻辑优先，UI 只负责显示和交互 |
| Presentation | HUD、背包界面、音频和特效 | 与核心玩法状态解耦 |
| Tests | 高风险规则验证 | 优先覆盖占格、合并、掉落和区域判定 |

核心工程原则：

- 静态配置与运行时状态分离，避免运行过程中修改原始资源。
- 主动武器和自动武器复用投射物、命中与伤害结算管线。
- 背包规则先实现为可独立测试的二维网格逻辑，再连接拖拽 UI。
- 敌人、投射物和掉落物采用对象池，面向大量单位场景控制性能开销。
- 表现层只订阅状态和事件，不保存真实玩法数据。

## 本地运行

1. 克隆仓库：

   ```bash
   git clone https://github.com/cpz66l/BackpackSurvivor.git
   ```

2. 在 Unity Hub 中选择“添加项目”，打开克隆目录下的 `BackpackSurvivor` 文件夹。
3. 建议使用 Unity `6000.3.20f1`，或使用兼容的 Unity 6 编辑器版本。
4. 打开以下原型场景：

   ```text
   Assets/BackpackSurvivor/Scenes/Run/01-Run.unity
   ```

5. 进入 Play Mode 运行当前原型。

## 个人职责与展示重点

本项目由个人独立规划并持续开发，主要覆盖：

- 游戏玩法构思、规则设计与 GDD 编写
- Unity 工程搭建、目录规划和版本管理
- 玩家控制、战斗、敌人、掉落与波次系统开发
- 网格背包的数据结构、交互规则与 UI 实现
- ScriptableObject 数据驱动与运行时状态管理
- 模块拆分、接口设计、性能优化与测试规划
- 原型资源整合、反馈效果调优和演示材料制作

希望通过该项目展示的不只是单个功能的实现，而是将一个个人玩法想法逐步转化为 **可维护、可扩展、可验证的完整游戏工程** 的能力。
