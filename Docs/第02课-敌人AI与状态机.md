### 核心代码：EnemyAI（第 2 课定稿版）

```csharp
using BS.GamePlay.Combat;
using UnityEngine;

namespace BS.GamePlay.Enemies
{
    [RequireComponent(typeof(CharacterController))]
    [RequireComponent(typeof(Health))]//组件契约
    public class EnemyAI : MonoBehaviour
    {
        //追击
        [SerializeField] private float moveSpeed = 3f;
        [SerializeField] private float viewRange = 10f;//视野范围
        //攻击
        [SerializeField] private float attackRange = 3f;
        [SerializeField] private float attackInterval = 3f;
        [SerializeField] private float contactDamage = 10f;
        [SerializeField] private float knockbackForce = 1f;

        private CharacterController cc;
        private Health health;
        private Transform playerTf;
        private Health playerHealth;
        private float attackTimer = 0f;

        private void Awake()
        {
            cc = GetComponent<CharacterController>();
            health = GetComponent<Health>();
        }
        private void Start()
        {
            //获取玩家信息
            GameObject player = GameObject.FindGameObjectWithTag("Player");
            if (player == null)
            {
                enabled = false;
                return;                     // 守卫子句的灵魂是 return
            }
            playerTf = player.transform;
            playerHealth = player.GetComponent<Health>();
            //订阅死亡事件
            health.OnDeath += Die;
        }

        private void Update()
        {
            if (playerTf == null || playerHealth.IsDead) return;
            Vector3 toPlayer = playerTf.position - transform.position;
            toPlayer.y = 0;                          // 水平距离，忽略 Y 轴
            float distance = toPlayer.magnitude;

            if (distance > viewRange) return;        // 视野外待机
            else if (distance > attackRange)         // 追击态
            {
                Quaternion lookTarget = Quaternion.LookRotation(toPlayer);
                transform.rotation = Quaternion.RotateTowards(
                    transform.rotation, lookTarget, 360f * Time.deltaTime);  // 必须乘 deltaTime
                cc.SimpleMove(toPlayer.normalized * moveSpeed);              // 自带重力
            }
            else                                     // 攻击态
            {
                attackTimer += Time.deltaTime;
                if (attackTimer >= attackInterval)
                {
                    attackTimer -= attackInterval;   // 减法计时，防时间漂移
                    DamageInfo info = new DamageInfo(
                        contactDamage, gameObject, playerHealth.Position, false, knockbackForce);
                    playerHealth.TakeDamage(info);
                }
            }
        }

        private void Die()
        {
            Debug.Log($"{gameObject.name} 被击杀");
            health.OnDeath -= Die;                   // 有 += 就有 -=
            Destroy(gameObject);                     // 第 3 课演进为 SetActive(false)
        }
    }
}
```

### 核心代码：HazardZone 中立阵营判定

```csharp
[SerializeField] private Faction targetFaction = Faction.Player;   // 作用对象字段化

private void OnTriggerEnter(Collider other)
{
    IDamageable target = other.GetComponent<IDamageable>();
    if (target == null) return;
    // 中立区不过滤阵营；非中立区只命中匹配阵营
    if (targetFaction != Faction.neutral && target.Faction != targetFaction) return;
    if (!targetsInZone.Contains(target)) targetsInZone.Add(target);
}
```

## 二、关键设计决策与思考

1. **最小状态机（FSM 胚胎）**：待机/追击/攻击三态由一个距离判断链切换。状态机的全部真相是"按条件切换行为分支"——状态少时 if-else 就是最优解，状态多了再抽正式 FSM。精英怪"潜行→突袭"、Boss 多阶段都是它的长大版。
2. **Awake/Start 分工铁律**：Awake 只碰自己（GetComponent 缓存），Start 才碰别人（找玩家、订阅事件）。Unity 保证所有 Awake 先于所有 Start，跨对象访问无初始化时序问题。
3. **查找缓存，禁止每帧 Find**：`FindGameObjectWithTag` 是全场景遍历，Start 里查一次缓存终身用。Find 类 API 进 Update = 性能自杀。
4. **RequireComponent 组件契约**：用声明消灭一整类运行时错误——挂脚本自动补齐依赖组件，GetComponent 永不落空。
5. **事件订阅闭环**：`health.OnDeath += Die` 把"死亡时做什么"挂进喇叭名单；Die 里先 `-=` 退订再销毁。**方法当参数传时不加括号**——`Die` 是菜谱，`Die()` 是现在炒菜。
6. **SimpleMove vs Move**：SimpleMove 自带重力、忽略垂直速度，平地生物首选；Move 不管重力需自己累加（做跳跃时才需要）。
7. **物理组件互斥**：CharacterController 自带碰撞体，旧 CapsuleCollider 必须删——两个碰撞体并存会被物理引擎乱推，新手经典灵异现象。
8. **配置化优于继承**：危险区差异（阵营/中立）是数据差异，做成 SerializeField 字段而非子类。差异是行为才谈继承，差异是数据就走配置。

## 三、踩过的坑（排错记录）

- **守卫子句没有 return**：`enabled = false` 只挡后续帧，当前 Start 继续执行，下一行 null.GetComponent 直接炸——**只打状态不阻断执行等于没守**。
- **RotateTowards 没乘 deltaTime**：第三参数是"每次调用最多转角"，写死 360 等于每帧 360°，帧率越高转身越快——帧率耦合的隐形版本。
- **IDE 乱加 using**：`Unity.Mathematics` 被自动导入且未使用——自动导入必须人工过目（继 NUnit 之后第二次）。
- **`GetComponent<Transform>()`**：Transform 是必有组件，直接用 `player.transform` 属性后门。
- **枚举成员小写**：`Faction.neutral` 应为 `Neutral`——公开成员一律 PascalCase。
- **"范围外不动却还在掉血"虚惊**：排查后发现伤害源是中立危险区而非敌人——多伤害源系统里，定位问题先分清"谁在打我"，DamageInfo 的 attacker 字段价值体现。

## 四、面试会怎么问

- **Q：敌人 AI 怎么实现的？** A：最小双态状态机，距离驱动切换；状态膨胀后会抽 FSM 或用行为树——能说出"为什么现在不用行为树"（过度设计）更加分。
- **Q：怎么找玩家的？为什么？** A：原型期 Tag 查找 + 缓存；知道局限（单玩家、全场景遍历）并能说出演进路径（注册表/依赖注入）。
- **Q：事件订阅为什么要退订？** A：发布者寿命 > 订阅者时，死引用导致 MissingReference 或 GC 泄漏；同生共死的自订阅可豁免，但习惯成对写。
- **Q：攻击计时器为什么 `-=` 不归零？** A：帧离散导致的零头会累积成时间漂移，减法保持长期频率准确。
- **Q：距离判断为什么忽略 Y 轴？** A：胶囊体高度差会让"水平贴脸"的目标永远算成 1 米开外；2.5D 判定一律投影到 XZ 平面。

## 五、习惯养成清单

- [x] 守卫子句必带 return；先守后做
- [x] 计时器一律 `+= deltaTime` / `-= interval`，不与帧率耦合
- [x] Find/GetComponent 只在外壳回调做一次，结果缓存
- [x] 事件 `+=` 与 `-=` 成对出现
- [x] 自动导入的 using 逐个人工确认
