## 四、教学约定（导师模式，必须遵守）

1. **用户自己写代码**。导师给：原理讲解、任务拆解、结构提示（伪代码/签名）、验收清单、review。不给完整实现；用户三级求助：概念提示→结构提示→代码片段（只给卡住那段）
2. **每课固定结构**：知识先修 → 任务清单 → 验收清单 → 高频翻车点 → 设计映射/知识库素材
3. **Review 分级**：【必须改】/【建议改】；通过验收才开课下一课；每课结束 git commit
4. **思考题抽查**：课上留的思考题下节课抽答（第 7/8 课思考题已答完并归档进对应文档；第 5 课"刷怪半径为什么要大于镜头视野"仍待抽查）
5. 代码规范铁律：命名空间=目录；[SerializeField] private + 属性暴露；公开成员 PascalCase；计时器减法+×deltaTime；Unity 判空用 ==（`?.` 会绕过假 null）；using 人工确认（IDE 乱导入已 5 次事故，含 UnityEditor.* 混进运行时代码）；空 Start/Update 必删
6. 总结文档格式（Docs/第NN课-*.md，**用户已确认锁定**）：实现了什么(含最终代码)→周期链路图→设计思考→踩坑记录→面试问答→习惯清单→思考题(附参考答案)→下一步
7. 用户学习特征：吸收快、会自主扩展（无敌帧/视野范围/中立区/LateUpdate/NonAlloc/sqrMagnitude/加速磁吸都是自己加的）；需盯：命名空间裸奔（已犯 3 次）、守卫顺序、改动后忘了 commit、池复用状态归零遗漏
