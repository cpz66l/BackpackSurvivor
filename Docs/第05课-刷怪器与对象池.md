### 核心代码 1：ObjectPool 完整周期

```csharp
public class ObjectPool : MonoBehaviour
{
    [SerializeField] private GameObject prefab;
    [SerializeField] private int prewarmCount = 30;
    private Queue<GameObject> idle = new Queue<GameObject>();
    private HashSet<GameObject> idleSet = new HashSet<GameObject>();   // 防重复归还

    private void Start()   // 预热：把扩容成本从战斗中挪到加载时
    {
        for (int i = 0; i < prewarmCount; i++)
        {
            GameObject obj = Instantiate(prefab);
            obj.SetActive(false);
            idle.Enqueue(obj); idleSet.Add(obj);
        }
    }

    public GameObject Get(Vector3 pos)
    {
        GameObject obj = idle.Count == 0
            ? Instantiate(prefab)          // 弹性扩容：首次归还时自动"入籍"
            : DequeueAndUntrack();
        obj.transform.position = pos;
        obj.SetActive(true);               // 触发 OnEnable：注册表登记 + 事件订阅
        IPoolable poolable = obj.GetComponent<IPoolable>();
        if (poolable != null)
        {
            poolable.SetPool(this);        // 发"回家车票"
            poolable.OnGetFromPool();      // 重置状态（满血/清零计时器）
        }
        return obj;
    }

    public void Return(GameObject obj)
    {
        if (obj == null || idleSet.Contains(obj)) return;   // 判空 + 防重复入队
        obj.GetComponent<IPoolable>()?.OnReturnPool();
        obj.SetActive(false);              // 触发 OnDisable：注册表注销 + 退订
        idle.Enqueue(obj); idleSet.Add(obj);
    }
}
```

### 核心代码 3：EnemyAI 的池化适配（生命周期对称的完整形态）

```csharp
private void OnEnable()  { TargetRegistry.Register(health); health.OnDeath += Die; }
private void OnDisable() { TargetRegistry.Unregister(health); health.OnDeath -= Die; }

private void Die()
{
    if (pool != null) pool.Return(gameObject);   // 有票回家
    else gameObject.SetActive(false);            // 无票（场景手摆）兜底退场
}

public void OnGetFromPool() { health.ResetToFull(); attackTimer = 0f; }
public void SetPool(ObjectPool p) => pool = p;
public void OnReturnPool() { }
```

## 二、关键设计决策与思考

1. **对象池的成本模型**：Instantiate/Destroy = CPU 分配 + GC 卡顿；池化 = 用内存换 CPU/GC，高频生灭对象（敌人/子弹/掉落物）必须池化。
2. **完整周期链路**：预热→Get（出队→定位→激活→SetPool→重置）→服役→Die→Return（判空→防重→回调→休眠→入队）→复生。Unity 生命周期（OnEnable/OnDisable）与池契约（SetPool/OnGet/OnReturn）两套回调交织对齐。
3. **IPoolable 的解耦价值**：池子不认识任何具体类型，只发三条指令；各对象自己实现重置逻辑——与 IDamageable 同一哲学的第三次应用，项目接口主线增至两条。
4. **SetPool 绑定 = 回家车票**：死亡由对象自己发起（事件驱动），归还账只有池子会算——出生时池子把自己的地址发给对象，死亡时对象持票自归还。绑定发生在 Get，终身有效。
5. **弹性扩容与最高水位**：池空现场 Instantiate 的对象首次归还即入籍，池容自动收敛于本局同屏峰值；预热的意义是把扩容成本挪出战斗，频繁扩容=预热估小的信号。
6. **池化三大"复活伦理"**：①状态重置（HP/计时器，否则会尸体复活）②订阅搬家（Start 一辈子跑一次，订阅必须挪 OnEnable/OnDisable）③注册表同步（OnEnable/OnDisable 天然兼容休眠语义——第 3 课的两行注册在池化时零改动兑现）。
7. **存活计数复用花名册**：`TargetRegistry.Count` 即场上敌人数，刷怪上限零新增数据结构——注册表的第三重价值。

## 三、踩过的坑（排错记录）

- **度与弧度**：`Random.Range(0,360)` 直接喂 `Mathf.Sin`——Sin/Cos 吃弧度，必须 `* Mathf.Deg2Rad`。周期函数让错误"碰巧能跑"，最危险。
- **`Equals(null)` 判空误区**：Unity 的"假 null"只有 `==` / `bool` 运算符能识别，`Equals` 做引用比较永远漏判——**Unity 对象判空只用 `== null`**。
- **UnityEngine.Pool 撞名**：IDE 自动导入与自带 `ObjectPool<T>` 同名冲突——自动导入第四次事故，最危险的一次。
- **Die 空池 NRE**：没经过池子出生的对象没有车票——`pool != null` 守卫 + 兜底退场路径。
- **刷怪器"算完不生成"**：位置算完忘了调用 Get/Instantiate——功能代码写完后自查"每个分支的终点动作是什么"。
- **Start 订阅丢失**：池化复用后 Start 不重跑，订阅没了第二次死亡无人收尸——订阅/登记只住 OnEnable/OnDisable。

## 四、面试会怎么问

- **Q：对象池怎么实现的？** A：Queue 存休眠对象 + HashSet 防重复归还；预热 + 弹性扩容；IPoolable 三钩子解耦重置逻辑；对象持池引用自归还。能画出完整周期链路图。
- **Q：为什么预热？预热量怎么估？** A：把 Instantiate 成本从战斗挪到加载；按同屏峰值估算，战斗中扩容次数=预热是否足够的量化信号。
- **Q：池化对象的状态怎么保证干净？** A：复位钩子集中在 OnGetFromPool（本项目敌人=ResetToFull+计时器清零，子弹=Initialize 重灌）；**最常见的池化 bug 就是状态泄漏**，防线是"所有可变状态必须在钩子里显式重置"。
- **Q：Unity 生命周期回调在池化下的语义变化？** A：Awake/Start 一辈子一次（放一次性初始化），OnEnable/OnDisable 每次激活/休眠都来（放登记/订阅），Destroy 被 SetActive(false) 取代。
- **Q：Unity 对象判空为什么不能用 Equals？** A：Unity 重载 == 实现"假 null"（对象已销毁、引用包装还在），Equals 是原生引用比较会漏判。
- **Q：为什么不用 Unity 自带 ObjectPool<T>？** A：本项目需要 MonoBehaviour 组件化配置（Inspector 拖 prefab、多场景区分实例）与 IPoolable 钩子定制；自带池适合纯 C# 对象的轻量场景。能说出取舍即可。

## 五、习惯养成清单

- [x] 高频生灭对象必问"池化了吗"，且新类从设计第一天就池化友好（Initialize 复位钩子）
- [x] 订阅/登记只放 OnEnable/OnDisable，Awake/Start 不碰跨对象
- [x] 三角函数先确认单位（弧度），随机数先确认区间开闭
- [x] 判空三选一：`== null` / `?.` / `??`，杜绝 Equals
- [x] 池子归还必防重（HashSet），取出必重置（IPoolable）
