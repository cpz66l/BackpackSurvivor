## 一、实现了什么

玩家静止不动也会自动开火：枪械子物体（Gun）独立转向最近敌人，枪口对准后发射子弹，子弹扫掠飞行、命中扣血、敌人死亡后自动切换下一个目标。

| 文件 | 职责 |
| --- | --- |
| `Combat/TargetRegistry.cs` | 全场可命中目标花名册：登记/注销/最近目标查询，武器不再遍历全场 |
| `Combat/Projectile.cs` | 子弹：扫掠式 SphereCast 命中检测、发射者层级豁免、零分配 NonAlloc、超程自毁 |
| `Combat/AutoWeapon.cs` | 自动武器：索敌 → 枪口转向（LateUpdate）→ 角度门槛 → 开火，全自动闭环 |
| `Enemies/EnemyAI.cs`（增量） | OnEnable/OnDisable 注册注销；死亡改为 SetActive(false)，为对象池铺路 |
| `Player/PlayerController.cs`（增量） | 转向改为只转 Model 子物体（bodyPivot），身体与枪口各自独立 |

### 核心代码 1：TargetRegistry

```csharp
using BS.GamePlay.Combat;
using System.Collections.Generic;
using UnityEngine;

public static class TargetRegistry   // 静态类：全局可查，原型期单场景方案（已知限制）
{
    private static readonly List<IDamageable> allTargets = new List<IDamageable>();
    public static int Count => allTargets.Count;

    public static IDamageable GetNearestTarget(Vector3 fromPos, float maxRange, Faction targetFaction)
    {
        float nearestDistance = float.MaxValue;
        IDamageable nearestTarget = null;
        for (int i = allTargets.Count - 1; i >= 0; i--)
        {
            if (allTargets[i] == null || allTargets[i].IsDead
                || allTargets[i].Faction != targetFaction) continue;
            float sqrDistance = (allTargets[i].Position - fromPos).sqrMagnitude;  // 免开方优化
            if (sqrDistance > maxRange * maxRange) continue;
            if (nearestDistance > sqrDistance)
            {
                nearestDistance = sqrDistance;
                nearestTarget = allTargets[i];
            }
        }
        return nearestTarget;
    }

    public static void Register(IDamageable t)
    {
        if (t == null || allTargets.Contains(t)) return;
        allTargets.Add(t);
    }
    public static void Unregister(IDamageable t)
    {
        if (!allTargets.Contains(t)) return;
        allTargets.Remove(t);
    }
}
```

### 核心代码 2：Projectile（命中检测节选）

```csharp
// 每帧扫掠检测：从当前位置向飞行方向投射本帧位移长度的球体射线
private bool TryGetFirstHit(Vector3 origin, Vector3 direct, float distance, out RaycastHit nearestHit)
{
    nearestHit = default;
    float nearestDistance = float.MaxValue;
    bool foundHit = false;
    Transform attackerRoot = attacker != null ? attacker.transform.root : null;

    int hitCount = Physics.SphereCastNonAlloc(           // NonAlloc：零 GC 分配
        origin, collisionRadius, direct, hitBuffer, distance,
        ~0, QueryTriggerInteraction.Ignore);             // 显式忽略 Trigger（防被危险区"吃掉"）

    for (int i = 0; i < hitCount; i++)
    {
        Collider hitCollider = hitBuffer[i].collider;
        if (hitCollider == null) continue;
        // 忽略发射者整套层级（玩家身体/Gun/FirePoint），防止子弹打自己
        if (attackerRoot != null && hitCollider.transform.root == attackerRoot) continue;
        if (hitBuffer[i].distance >= nearestDistance) continue;
        nearestDistance = hitBuffer[i].distance;
        nearestHit = hitBuffer[i];
        foundHit = true;
    }
    return foundHit;
}
// 命中后：GetComponentInParent<IDamageable>() → 阵营匹配 → TakeDamage(hit.point 精确受击点) → 销毁
```

### 核心代码 3：AutoWeapon（转向与开火节选）

```csharp
private void LateUpdate()   // 所有 Update 之后执行：Player 转完身体，再写枪口世界旋转
{
    currentTarget = TargetRegistry.GetNearestTarget(firePoint.position, maxRange, targetFaction);
    if (currentTarget == null) { attackTimer = 0f; return; }   // 无目标保持就绪

    Vector3 direction = currentTarget.Position - firePoint.position;
    if (direction.sqrMagnitude < 0.0001f) return;

    Quaternion targetRotation = Quaternion.LookRotation(direction, Vector3.up);
    aimPivot.rotation = Quaternion.RotateTowards(
        aimPivot.rotation, targetRotation, rotationSpeed * Time.deltaTime);  // 只转 Gun，不转 Player

    attackTimer += Time.deltaTime;
    if (attackTimer < attackInterval) return;
    // 枪口未对准不开火：避免视觉朝向与真实弹道撕裂
    if (Quaternion.Angle(aimPivot.rotation, targetRotation) > fireAngleTolerance) return;

    attackTimer -= attackInterval;
    Fire((currentTarget.Position - firePoint.position).normalized);
}

private void Fire(Vector3 direction)
{
    // 代码路线生成子弹：空物体 + AddComponent（灰盒期免 Prefab 依赖）
    GameObject bulletObj = new GameObject("bullet");
    bulletObj.transform.position = firePoint.position;
    Projectile bullet = bulletObj.AddComponent<Projectile>();   // 此刻 Awake 立即执行
    bullet.Initialize(projectileSpeed, damage, targetFaction, maxDistance, direction, 0f, gameObject);
}
```

## 二、关键设计决策与思考

1. **注册表模式**：索敌查询 O（场内敌人数） 替代 O（全场对象数）。敌人"变化时自登记"分摊维护成本，武器只问花名册——"变化时更新缓存"第三次复用（危险区列表 → 注册表）。
2. **生命周期对称（OnEnable/OnDisable）**：登记注销与对象激活状态对齐。对象池用 SetActive 休眠/唤醒，Awake/OnDestroy 一辈子只跑一次覆盖不了；OnDisable 在真销毁时也会补发，注销永不漏。
3. **扫掠式碰撞检测**：高速子弹一帧飞 0.33m，离散帧间直接"穿过"敌人（隧道效应）。每帧从旧位置向新位置打 SphereCast，连续路径检测，再快不漏。
4. **零分配 NonAlloc**：`SphereCastNonAlloc` + 预分配 `RaycastHit[8]` 缓冲区，命中结果复用内存——子弹是高频对象，GC 分配必须前置消灭。
5. **执行顺序分层（Update/LateUpdate）**：身体朝向（PlayerController·Update）与枪口朝向（AutoWeapon·LateUpdate）写入顺序分层，后写覆盖先写，渲染取最终值——解决"父物体逻辑覆盖子物体"。
6. **视觉与弹道合一（开火角度门槛）**：枪口旋转需要时间，未对准不开火，消除"枪没转过来子弹先飞"的撕裂感。
7. **发射者层级豁免**：`transform.root` 按结构豁免整套层级，优于按 Tag 名单豁免——新增挂载物零维护。
8. **Instantiate 是复印机不是创造机**：必须有模板才能克隆；灰盒期用 `new GameObject + AddComponent` 代码路线，Projectile 视觉代码自产，零资产依赖。
9. **静态类的已知限制**：static 数据寿命=应用寿命，跨场景残留"假 null"。单场景原型可用，多场景时升级场景级服务——全局状态爽完总要还账。

## 三、踩过的坑（排错记录）

- **Instantiate(null)**：声明变量 ≠ 创建对象，复印机没有原件直接罢工——分清"声明/创建/克隆"三个动作。
- **父子朝向"覆盖"**：真相不是父子关系，而是写入顺序——PlayerController 在 Update 转身体，AutoWeapon 若也在 Update 转枪则同帧竞争；LateUpdate 后写者胜。
- **子弹被隐形物体"吃掉"**：Unity 射线默认命中 Trigger，危险区这种无形 Trigger 会拦截弹道——显式 `QueryTriggerInteraction.Ignore`。
- **子弹"偏差很大"**：枪口未对准就开火，视觉与弹道分离——开火角度门槛修复。
- **命中不掉血**：复制敌人后 Health 阵营忘了配 Enemy——数据配置类 bug，日志里 attacker/faction 一打一验便知。
- **残留债务**：`using System.Runtime.InteropServices;` 未清（IDE 乱导入第三次）、TargetRegistry 未包命名空间——记入第 6 课 hygiene。

## 四、面试会怎么问

- **Q：自动索敌怎么做的？** A：目标注册表（TargetRegistry）集中管理，敌人 OnEnable/OnDisable 自登记注销；武器按射程+阵营+最近优先查询，不遍历全场。进阶可讲 tick 降频（GDD 20.1 的 5-10Hz 方案）。
- **Q：子弹命中怎么实现的？会穿透吗？** A：扫掠式 SphereCast 解决隧道效应；当前命中即毁，穿透只需在命中后继续飞行并记录已命中名单——说出扩展点比实现过更重要。
- **Q：NonAlloc 系列 API 解决什么问题？** A：Physics.RaycastAll 等每次返回新数组产生 GC 分配，高频调用引发 GC 峰值卡顿；NonAlloc 复用预分配缓冲区，零分配。
- **Q：为什么用 LateUpdate？** A：需要在所有 Update 完成后、渲染前执行，保证读到本帧最终姿态——相机跟随（Cinemachine 内部也是 LateUpdate）同理。
- **Q：静态类做管理器有什么问题？** A：跨场景状态残留、依赖关系不可见、无法多实例、测试困难；适用面=单场景原型/纯工具，正式项目用场景级服务或 DI。
- **Q：为什么登记用 OnEnable 而不是 Awake？** A：对象池 SetActive 复用场景下 Awake 一辈子只跑一次，OnEnable/OnDisable 每次激活/休眠都触发，且 OnDisable 在销毁时也补发——生命周期对称。
