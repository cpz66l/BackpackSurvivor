# 第 1 课：伤害管线（IDamageable / Health）+ 区域危险（HazardZone）

> 项目：《背包幸存者》Backpack Survivor | 日期：2026-07-20 | 阶段：V0.1 战斗核心原型 · 第 1 课

## 一、实现了什么

搭建了全项目共用的战斗地基——伤害结算管线，并用一个危险区验证了它的可用性。

| 文件 | 职责 |
| --- | --- |
| `Combat/IDamageable.cs` | 伤害管线契约：`IDamageable` 接口（TakeDamage / Position / IsDead / Faction）、`DamageInfo` 伤害数据包（伤害值/攻击者/命中点/暴击/击退，readonly 不可变）、`Faction` 阵营枚举 |
| `Combat/Health.cs` | 通用生命组件：实现 IDamageable，扣血守卫（IsDead）、Clamp 钳制，对外广播 `OnDamaged` / `OnDeath` 事件；玩家、敌人、未来一切可受伤单位共用 |
| `Zones/HazardZone.cs` | 危险区组件：Trigger 进出差额维护目标缓存，固定 tick（0.5s）对区内玩家阵营目标结算持续伤害 |

运行效果：玩家走进危险区每 0.5 秒掉 5 血（10 秒伤），走出即停；死亡后广播死亡事件且不再受伤。

## 二、关键设计决策与思考

1. **接口抽象（依赖倒置）**：武器/危险区只认 `IDamageable`，不认 `Health`。伤害来源不需要知道打的是谁——以后宝箱、油桶、Boss 实现同一接口即可进入战斗管线，调用方零改动。
2. **DamageInfo 数据包**：伤害结算的入参不是裸 float，而是包含攻击者/命中点/暴击/击退的数据包，且 readonly 不可变——创建后不允许第三方篡改，为暴击、击退、飘字、击杀归属预留了扩展。
3. **事件驱动（发布-订阅）**：Health 不持有血条/飘字的引用，只广播 `OnDamaged`/`OnDeath`；表现层"谁关心谁订阅"。符合"表现层不保存玩法数据、只订阅状态"的工程原则。
4. **帧率解耦**：持续伤害用固定 tick 结算而非每帧扣血——不同帧率机器上数值一致；每跳伤害 = 秒伤 × tick 间隔，保证"配置语义（秒伤）与实现细节（结算频率）解耦"。
5. **事件驱动缓存**：Enter/Exit 维护目标列表，tick 时直接遍历，避免每 tick 全场景物理查询（Physics.OverlapSphere）。"变化时更新缓存"优于"需要时全量查询"。
6. **倒序遍历清理**：循环中删除元素必须倒序；目标为 null（已销毁）或 IsDead 时从缓存剔除，防止 MissingReferenceException。
7. **抽象时机（Rule of Three）**：讨论过"是否先写 Zone 基类"——结论是差异只有数据（阵营/时长/归属）而无行为时，优先字段配置化而非继承；抽象等第二个真实案例出现后再提炼。
8. **接口最小暴露的取舍**：为让伤害方拿到受击点坐标，接口新增 `Vector3 Position` 属性，而不是暴露整个 Transform——接口只开放"恰好够用"的能力。

## 三、踩过的坑（排错记录）

- **GBK 编码**：中文系统下 VS 默认 ANSI 保存 → 统一以 `.editorconfig`（`charset = utf-8-bom`）强制规范，根治编码漂移。
- **NUnit.Framework.List 撞名**：IDE 自动导入命名空间选错目标，`List<T>` 解析到 NUnit 非泛型类 → 玩法代码禁止 using 测试框架命名空间；自动导入必须人工确认。
- **0 值暴击炸弹**：未赋值的 `criticalMultiplier` 默认 0，暴击反而零伤害 → 未接线的字段不上车；且暴击倍率属于攻击方职责，Health 只执行不计算。
- **null 攻击者**：环境伤害无攻击者，`attacker.name` 直接 NRE → 一切外部输入做 null 兜底（`attacker?.name ?? "环境"`）。
- **序列化字段名即存储键**：Unity 按字段名序列化，改名丢 Inspector 配置 → 字段命名一次到位。

## 四、展示的能力（面试可讲）

- 用接口做系统间解耦（SOLID 之依赖倒置）的实战落地
- 发布-订阅事件驱动架构在 Unity 战斗管线中的应用
- Unity 物理系统：Trigger 判定、组件缓存、生命周期回调
- 数值设计工程化：帧率无关结算、配置与实现解耦
- 防御式编程：守卫子句、null 兜底、失效引用清理
- 工程规范：命名空间=目录结构、最小暴露（[SerializeField] private）、.editorconfig 统一编码
