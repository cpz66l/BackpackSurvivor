## 一、这节课实现了什么

杀怪掉落系统，完整链路：

```
敌人死亡 → 掷骰决定掉什么 → 从对象池取出掉落物 → 按稀有度变色 → 超时自动回收
```

四个新脚本 + 敌人侧一处接线：

| 脚本 | 命名空间 | 职责 |
|------|---------|------|
| `LootTableData` | BS.Data | ScriptableObject 配置资产：掉落条目数组（预制体 + 稀有度 + 权重）+ 总权重属性 |
| `LootRoller` | BS.GamePlay.Loot | **纯 C# 类**（不继承 MonoBehaviour）：权重随机 + 保底计数，核心算法 |
| `LootManager` | BS.GamePlay.Loot | 场景单例式入口：持有掉落表和对象池，对外只暴露 `TrySpawnDrop(position)` |
| `DropItem` | BS.GamePlay.Loot | 掉落物本体：IPoolable、自转、按稀有度变色、超时自动回收 |
| `EnemyAI`（改） | BS.GamePlay.Enemies | 死亡事件 Die() 里调用 `lootManager.TrySpawnDrop(health.Position)` |

## 二、最终代码

### LootTableData.cs —— 掉落表配置资产

```csharp
using System;
using System.Collections.Generic;
using UnityEngine;

namespace BS.Data
{
    /// <summary>
    /// 掉落表配置资产，用于定义一组掉落物品及其权重和稀有度。
    /// 与 GDD 11.3 稀有度等级对齐。
    /// </summary>
    [CreateAssetMenu(fileName = "NewLootTable", menuName = "BackpackSurvivor/LootTable")]
    public class LootTableData : ScriptableObject
    {
        //嵌套类,掉落物条目
        [Serializable] public class LootEntry
        {
            [Tooltip("掉落的预制体")]
            public GameObject dropPrefab;

            [Tooltip("物品稀有度")]
            public Rarity rarity;

            [Tooltip("权重值（越大掉落概率越高）")]
            public int weight;
        }

        [Tooltip("所有可能的掉落条目")]
        public LootEntry[] entries;

        /// <summary>
        /// 总权重（所有条目权重之和）。用于概率计算。
        /// </summary>
        public int TotalWeight
        {
            get
            {
                int total = 0;
                if (entries != null)
                {
                    foreach (LootEntry entry in entries)
                    {
                        if (entry != null && entry.weight > 0) // 忽略无效或零权重条目
                            total += entry.weight;
                    }
                }
                return total;
            }
        }
    }

    public enum Rarity
    {
        Common,     // 白色
        Uncommon,   // 绿色
        Rare,       // 蓝色
        Epic,       // 紫色
        Legendary   // 金色
    }
}
```

### LootRoller.cs —— 权重随机 + 保底（纯类）

```csharp
using BS.Data;
using System;
using static BS.Data.LootTableData;

namespace BS.GamePlay.Loot
{
    /// <summary>
    /// 带保底机制的掉落掷骰器。
    /// </summary>
    public class LootRoller
    {
        private int pityCount;
        private int pityThreshold;

        public LootRoller(int pityThreshold)//构造函数
        {
            this.pityThreshold = pityThreshold;
            pityCount = 0;
        }

        /// <summary>
        /// 根据掉落表进行一次抽取，自动处理保底逻辑。
        /// </summary>
        /// <param name="table">掉落表资产</param>
        /// <returns>选中的掉落条目，若无有效条目则返回 null</returns>
        public LootEntry Roll(LootTableData table)
        {
            // 若条目为空或无效，直接返回 null
            if (table == null || table.entries == null || table.entries.Length == 0)
                return null;

            //触发保底阶段
            if (pityCount >= pityThreshold)
            {
                //获得从稀有度≥蓝 的条目
                LootEntry[] eligible = Array.FindAll
                    (table.entries, e => e != null && e.weight > 0
                    && (int)e.rarity >= (int)Rarity.Rare);

                if (eligible.Length > 0)
                {
                    LootEntry selected = PickByWeight(eligible);
                    if (selected != null)
                    {
                        pityCount = 0;
                        return selected;
                    }
                }
            }

            //全表抽取
            LootEntry result = PickByWeight(table.entries);
            //中蓝以上保底也清零
            if ((int)result.rarity >= (int)Rarity.Rare) pityCount = 0;
            //没中则奖池累计
            else pityCount++;

            return result;
        }

        /// <summary>
        /// 从给定条目数组中按权重随机选择一个。
        /// </summary>
        private LootEntry PickByWeight(LootEntry[] entries)
        {
            // 计算总权重
            int total = 0;
            foreach (var e in entries)
            {
                if (e != null && e.weight > 0)
                    total += e.weight;
            }

            if (total <= 0)
                return null; // 无有效权重

            //随机掷点
            int roll = UnityEngine.Random.Range(0, total);
            int accum = 0;
            foreach (var e in entries)
            {
                if (e != null && e.weight > 0)
                {
                    accum += e.weight;
                    if (roll < accum)
                    {
                        return e;
                    }
                }
            }
            return null;//理论上不会到达
        }
    }
}
```

### LootManager.cs —— 掉落入口

```csharp
using BS.Core;
using BS.Data;
using UnityEngine;
using static BS.Data.LootTableData;

namespace BS.GamePlay.Loot
{
    public class LootManager : MonoBehaviour
    {
        [SerializeField] private LootTableData lootTable;//获取一组掉落物配置表
        [SerializeField] private ObjectPool dropPool;
        [SerializeField] private int pityThreshold = 10;//保底数

        private LootRoller lootRoller;

        void Start()
        {
            lootRoller = new LootRoller(pityThreshold);
        }

        public void TrySpawnDrop(Vector3 position)
        {
            //从这组table配置表中获取掉落物条目
            LootEntry entry = lootRoller.Roll(lootTable);
            if (entry == null) return;
            DropItem dropItem = dropPool.Get(position).GetComponent<DropItem>();
            dropItem.Initialize(entry.rarity);
        }
    }
}
```

### DropItem.cs —— 掉落物本体

```csharp
using BS.Core;
using BS.Data;
using UnityEngine;

namespace BS.GamePlay.Loot
{
    public class DropItem : MonoBehaviour, IPoolable
    {
        [SerializeField] private float rotateSpeed = 180f;
        [SerializeField] private float survivalTime = 10f;
        [SerializeField] private Rarity rarity = Rarity.Common;

        private float survivalTimer = 0f;
        //对象池
        private ObjectPool pool;
        public void SetPool(ObjectPool p) => pool = p;

        //视觉组件
        private GameObject _visualModel;//Loot模型
        private Renderer rd;

        private void Awake()
        {
            //创建简易Loot模型
            _visualModel = GameObject.CreatePrimitive(PrimitiveType.Sphere);
            _visualModel.transform.SetParent(transform);
            _visualModel.transform.localScale = Vector3.one * 0.4f;
            _visualModel.transform.localPosition = Vector3.zero;

            //移除碰撞器避免干扰射线检测 (本身不需要物理碰撞)
            Destroy(_visualModel.GetComponent<Collider>());

            rd = _visualModel.GetComponent<Renderer>();
            if (rd != null)
            {
                rd.material.color = Color.yellow;
            }
        }

        private void Update()
        {
            transform.Rotate(Vector3.up * rotateSpeed * Time.deltaTime);
            survivalTimer += Time.deltaTime;
            if (survivalTimer >= survivalTime)
            {
                Recycle();
            }
        }

        public void Initialize(Rarity rarity)
        {
            this.rarity = rarity;
            switch (rarity)
            {
                case Rarity.Common:
                    rd.material.color = Color.white;                 // 白
                    break;
                case Rarity.Uncommon:
                    rd.material.color = Color.green;                 // 绿
                    break;
                case Rarity.Rare:
                    rd.material.color = Color.blue;                  // 蓝
                    break;
                case Rarity.Epic:
                    rd.material.color = new Color(0.6f, 0.2f, 0.9f); // 紫
                    break;
                case Rarity.Legendary:
                    rd.material.color = new Color(1f, 0.84f, 0f);    // 金
                    break;
            }
        }

        private void Recycle()
        {
            if (pool != null) pool.Return(gameObject);
            else Destroy(gameObject);//（无池兜底）
        }

        public void OnGetFromPool()
        {
            survivalTimer = 0f;
        }

        public void OnReturnPool()
        {
        }
    }
}
```

### EnemyAI.cs —— 死亡接线（只列改动部分）

```csharp
// 新增字段
private LootManager lootManager;

private void Die()
{
    lootManager.TrySpawnDrop(health.Position);   // ★ 必须在 pool.Return 之前
    //防御，防止忘设pool，或者是没经过池子的敌人
    if (pool != null) pool.Return(gameObject);
    else gameObject.SetActive(false);
}

public void OnGetFromPool()
{
    health.ResetToFull();
    attackTimer = 0f;
    lootManager = FindAnyObjectByType<LootManager>();  // 给字段赋值，无类型前缀
}

public void OnReturnPool()
{
    // 留空：掉落只挂在 Die，绝不挂在这里
}
```

## 三、脚本之间的关系与完整周期链路

### 依赖关系图

```
EnemyAI ──调用──▶ LootManager ──拥有──▶ LootRoller ──读取──▶ LootTableData（SO资产）
                      │
                      │ Get(position) / Initialize(rarity)
                      ▼
                 ObjectPool ⇄ DropItem（IPoolable）
```

- **LootTableData**：纯数据，不引用任何人，被所有人读（策划的表）
- **LootRoller**：纯算法，只依赖 LootTableData 的数据结构，不碰 Unity 场景（可单元测试）
- **LootManager**：组装者——把表喂给 Roller，把结果喂给对象池
- **DropItem**：被动者——不主动找任何人，等 LootManager 来 Initialize
- **EnemyAI**：触发者——只知道"死了要喊 LootManager 一声"，不知道掷骰细节

### 一次掉落的完整周期链路

```
① 敌人 Health 归零
        │ 触发 OnDeath 事件
② EnemyAI.Die()
        │ lootManager.TrySpawnDrop(health.Position)   ← 受伤者坐标，不是 Zone 坐标
③ LootManager：lootRoller.Roll(lootTable)
        │ ├─ pityCount >= 阈值？→ 只在 ≥Rare 条目里权重抽，抽中清零
        │ └─ 否则全表权重抽：中 ≥Rare 清零，没中 pityCount++
④ dropPool.Get(position)
        │ ├─ 池里有存货 → 激活旧对象，SetPool + OnGetFromPool（survivalTimer 清零）
        │ └─ 池空 → 现场 Instantiate，同样走 SetPool + OnGetFromPool
⑤ dropItem.Initialize(entry.rarity)
        │ switch 按稀有度设置 renderer.material.color
⑥ DropItem.Update() 每帧：自转 + 计时
        │ survivalTimer >= survivalTime
⑦ Recycle() → pool.Return(gameObject) → SetActive(false)
        │ 对象回到池里，等下一次 ④ 被取走
```

**关键认知：池子里的对象是"循环服役"的**—— Instantiate 只发生在池空的时候，一旦池子里有了对象，它们就在 ④→⑦ 之间无限循环，再也不会 Destroy。

## 四、设计思考（这节课真正学的东西）

### 1. 为什么 LootRoller 是纯 C# 类，不继承 MonoBehaviour？

算法不需要场景、不需要 GameObject、不需要生命周期回调。纯类意味着：
- 可以在编辑器外写单元测试（`new LootRoller(10)`，跑一万次 Roll 统计概率分布）
- 不占用 Unity 的组件开销
- 职责单一：它只回答"这次掉什么"，不关心掉在哪、长什么样

**这是分层思维的体现：数据（SO）/ 算法（纯类）/ 表现（MonoBehaviour）三层分离。**

### 2. 权重随机为什么用"累加区间"而不是逐条判概率？

错误直觉：对每个条目掷一次骰子判中不中。问题是概率加起来不一定等于 1，可能全不中或中多个。

正确做法：把权重铺在数轴上——`[0, w1) [w1, w1+w2) ...`，掷一个 `[0, total)` 的随机点，落在谁的区间就选谁。**一次掷骰，必中且只中一个，概率天然归一。**

### 3. 保底机制的本质

pityCount 是"玩家运气欠条"：连续没中 Rare 以上就记账，账满了强制从稀有池里抽一张，然后销账。注意两个清零路径——保底触发清零、自然抽中也清零，少一个都会出 bug。

### 4. 掉落时机为什么必须在 pool.Return 之前？

Return 会 SetActive(false) 休眠对象。休眠后才生成掉落物，"从哪里掉出来"的位置语义就错了。更广的规则：**事件的结果（掉落）要在事件当事人（敌人）还"活着"时结算完。**

### 5. 颜色硬编码 switch 是数据驱动的雏形

策划调色要改代码 → 进阶方案是把"稀有度→颜色"做成 RarityConfig SO，代码只查表。知道演进方向即可。

## 五、踩坑记录

| 坑 | 根因 | 教训 |
|----|------|------|
| CS0120 编译错，`Die()` 里找不到 lootManager | `LootManager lootManager = FindAnyObjectByType<>()` 声明的是**局部变量**，方法结束即销毁 | **有类型名 = 新建局部变量；无类型名 = 给已有变量赋值**。要跨方法用，必须声明类级字段 |
| 一只怪掉两个球 | Die() 里调一次掉落，OnReturnPool 里又调一次，而 pool.Return 会自动触发 OnReturnPool | **掉落是"死亡"事件的结果，不是"回池"的结果**。副作用只挂一个事件源 |
| NRE 风险：`entry.rarity` | Roll() 在空表时返回 null，调用方没判空 | **别人的返回值，先用再信**。`if (entry == null) return;` 放方法最前 |
| 编译警告 CS0414：survivalTime 未使用 | LootManager 和 DropItem 里重复定义了同一个概念 | 同一职责的字段只留一处，警告不清理会掩盖真问题 |

## 六、面试问答

**Q：权重随机怎么实现？**
A：总权重求和 → `[0, total)` 掷点 → 累加区间命中。一次掷骰保证必中且只中一个，概率自动归一。时间复杂度 O(n)，够用；n 很大可以上前缀和 + 二分，O(log n)。

**Q：保底机制怎么设计？**
A：维护 pityCount，未中稀有则 +1，达到阈值后下一次只在稀有池抽，抽中（或自然中稀有）清零。要注意两个清零路径都要有。

**Q：掉落系统的对象为什么用对象池？掉落后不是销毁就行吗？**
A：幸存者类游戏同屏掉落物几十上百个，Instantiate/Destroy 会造成 GC 峰值卡顿。池化后对象循环服役，Instantiate 只在池空时发生。

**Q：MonoBehaviour 和纯 C# 类怎么选？**
A：需要 Unity 生命周期、场景交互、Inspector 配置的用 MonoBehaviour；纯算法、纯数据用普通类。掉落掷骰器做成纯类可以直接单元测试概率分布。

**Q：ScriptableObject 在这套系统里扮演什么角色？**
A：数据资产。掉落表是策划可调的配置，和代码解耦——改权重不用重新编译，同一套代码可以挂多张表（普通怪表、精英怪表、Boss 表）。

## 七、习惯清单（大厂日常）

- [x] 局部变量 vs 成员字段：跨方法使用的引用一律类级字段
- [x] 副作用只挂一个事件源（掉落只挂 Die，不挂 OnReturnPool）
- [x] 公共方法入口先判空（entry == null 直接 return）
- [x] 编译警告不过夜（CS0414 当场清理）
- [x] 事件结算在对象休眠前完成（TrySpawnDrop 先于 pool.Return）
- [x] 枚举穷举 switch，每个 case 有 break
- [x] 参数命名不用下划线前缀（那是私有字段的惯例）
